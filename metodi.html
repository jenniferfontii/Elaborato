Gestione di più siti web su un solo web server

come avviene e possibili soluzioni

Possono coesistere più siti su un solo web server?

Nella comunicazione appena vista un host comunica con un web server ospitante un solo sito web, è però possibile che sul server coesistano più siti. In questi casi è necessario intraprendere strade diverse da quella vista precedentemente in quanto più siti sullo stesso server avrebbero sia la stessa porta che lo stesso IP e questa omonimia causerebbe disagi nella comunicazione.

Metodi:

Ip Based

Il metodo IP-based consiste nel virtualizzare tanti host quanti i siti web che si intende ospitare sul server, assegnando a ciascuno un ip diverso. Utilizzando l’IPV4 questo metodo è altamente sconsigliato in quanto gli indirizzi versione 4 sono scarsi inoltre sarebbe necessaria una potenza di calcolo notevole per virtualizzare un numero alto di host. Il discorso cambia se si utilizza l’IPV6, la potenza di calcolo richiesta resta notevole ma la vastità di indirizzi versione 6 è molto più ampia di quella degli IPV4 quindi si incontrerebbero meno problemi.

Port Based

Un altro metodo è il Port-based, consiste nel dare una porta diversa a ciascun dominio mantenendoli sullo stesso server e avendo così un solo IP comune. Questo metodo è quasi completamente inutilizzato perché utilizzando una porta not well-known presuppone che l’utente finale oltre al nome di dominio sia a conoscenza anche della porta utilizzata, visto che sarebbe obbligatorio esplicitarla nella barra di ricerca. Se decidessimo di hostare due siti HTTP (porta 80) sullo stesso server utilizzando la soluzione Port-Based saremmo costretti a cambiare porta ad uno dei due siti (es porta 81) dunque anche ad esplicitare la porta nella barra di ricerca in quanto non convenzionale. L’ URL diventerebbe così www.esempio.com:81.

Name Based

Diversamente dai metodi visti precedentemente il metodo name-based permette ai siti ospitati sul server di condividere la stessa porta e lo stesso IP, avendo così tutti lo stesso socket. Il name-based per differenziare i vari siti si serve infatti del loro nome, come si è visto precedentemente la richiesta HTTP possiede un campo “host” in cui specifica il nome del sito che si desidera raggiungere. Proprio grazie a questo campo si è in grado di distinguere più siti pur aventi lo stesso socket. Il metodo Name-based presenta quindi diversi vantaggi, per esempio più siti possono coesistere usando lo stesso IP e restando sulla porta predefinita, per questo è il metodo più usato. Molti web server su cui abbiamo lavorato possono essere impostati come Name Based, tra cui Apache su Xampp, Tomcat e Altervista.
