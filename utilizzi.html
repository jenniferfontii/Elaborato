Utilizzi della tecnica Name Based

in diversi campi

Name Based su Xampp Apache

Appena installato, su Xampp esiste un solo dominio: http://localhost che corrisponde alla cartella C:\xampp\htdocs. Una soluzione possibile è quella di caricare ogni nuovo sito nella cartella htdocs e vederlo tramite l’indirizzo http://localhost/NomeSito, tuttavia questo approccio presenta diversi difetti come:

  • L’obbligo di lavorare esclusivamente su una cartella e creare quindi disordine.
  • I percorsi relativi alla cartella root potrebbero riscontrare problemi.
  • Il percorso del sito sarà http://localhost/NomeSito al posto di un semplice http://NomeSito

A questi problemi viene posto rimedio decidendo di creare diversi domini con host virtualization Name-based.

Name Based su Altervista

Gran parte dei siti che visitiamo ogni giorno sono ospitati su un Web Server tramite la tecnica name based. Un esempio che abbiamo incontrato durante l’anno è il servizio di Hosting “Altervista”.

Hostando un proprio sito su altervista si nota che il nome di dominio suggerito dal sito è così composto:

Analizzando il nome del dominio si nota una parte “variabile”, ovvero il nome del dominio che viene scelto da noi tra quelli disponibili durante la registrazione, e da una parte “fissa”: altervista.org/ che servirà per non creare conflitti con domini già esistenti nel web con nomi di dominio uguali a quello scelto da noi. Si può quindi presupporre che altervista utilizzi la tecnica name based nel suo servizio di hosting web, possiamo infatti escludere il metodo port-based perchè non è necessario specificare la porta di destinazione nell’URL. Il dubbio resta quindi tra IP-based e name based ma considerando l'efficienza del secondo e la struttura di nomenclatura offerta da Altervista è molto probabile che sia in uso il name-based virtual hosting.

Name Based su Tomcat

Un altro esempio di web server che utilizza la tecnica di virtual hosting Name-based è apache tomcat. Nel corso dell’anno scolastico abbiamo più volte usato tomcat per creare e gestire servlet Java. Tomcat, tuttavia, funziona diversamente rispetto ad apache. Infatti, nel nostro caso non necessita di particolari configurazioni.

Name Based su Microsoft Server

Alcuni servizi di hosting propongono Microsoft server al posto dei più comuni server Linux. Per fornire i servizi di web hosting è necessario configurare ed installare il componente IIS (internet information services). Al contrario del virtual hosting su server Linux questo risulta più intuitivo e user friendly in quanto tutti i passaggi si possono svolgere dalla classica interfaccia grafica di windows.
